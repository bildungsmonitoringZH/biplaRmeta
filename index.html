Package installieren

Das Package biplaRmeta setzt ein Element voraus:

  1. aktuelles und personalisiertes .Rprofile im R_USER Verzeichnis
    • Vorlage und Installationsanweisung verfügbar unter https://github.com/bildungsmonitoringZH/Rconfig

Zur Installation gibt es zwei Varianten:

  1. Die jeweils aktuellste Version des Packages ist zur Installation auf dem G: abgelegt
install.packages("G:/03_Arbeiten/Projekte/Statistikprogramm R/packages/biplaRmeta_1.0.zip", repos = NULL, type = "source")
  1. Das Package kann via RStudio/git in den lokalen Rworkspace geklont und von da installiert werden.
# cd ~/Documents/Rworkspace
# git clone https://www.dev.statistik.zh.ch:3000/KTZH-Bildung/biplaRmeta.git ./biplaRmeta
devtools::install(dependencies = FALSE, build_vignettes = TRUE)

1 Was sind Daten?

Die hier präsentierte Konzeption von Metainformationen für Daten ist grundsätzlich nicht an einen spezifischen Typ von Datensatz gekoppelt. Sie ist so allgemein und generisch gehalten, dass sie eine breite Palette von Formen, Typen oder Arten von strukturierten Daten zu beschreiben vermag.

Trotzdem steht eine spezifische Form von Daten als Vorstellung bei der Konzeptentwicklung und in den Beispielen im Vordergrund. Diese Daten sind repräsentiert in Tabellenform. Dabei repräsentiert eine Spalte ein Merkmal und eine Zeile einen Fall oder eine Gruppe von Fällen. Die Ausprägung eines Merkmals in einem Fall wird als Wert bezeichnet.

2 Was sind Metainformationen?

Metainformation kann vieles sein! Im Package biplaRmeta wird einerseits semantisch und anderererseits technisch definiert, welche Informationen als Metainformationen einer Dateneinheit gelten.

  1. Semantik der Metainformationen: Welche Elemente haben Metainformationen? Was sagen die einzelnen Elemente über die Daten aus?
  2. Abbildung der Metainformationen: Welche Struktur haben Metainformationen? Welche Elemente von Metainformationen werden (technisch) wie dargestellt?

Die Struktur der Metainformationen ist definiert durch 11 sogenannte Felder:

field swissdata opendata.swiss
title title dct:title
source_name source_name dct:publisher
source_url source_url dcat:accessURL
issued NA dct:issued
modified updated_utc dct:modified
units units NA
aggregate aggregate NA
dim_order dim_order NA
hierarchy hierarchy NA
labels labels NA
description details dct:description

Die elf Felder sind inspiriert durch das Metadaten-Format, das für das Open Government Data Switzerland project entwickelt wurde, sowie durch einen Vorschlag von @christophsax and @mbannert zur Verbesserung der Maschinenlesbarkeit von öffentlich verfügbaren Daten unter der Bezeichnung SwissData.

3 Semantik

Jedes der 11 Felder enthält spezifische, inhaltlich klar definierte Informationen über Daten.

3.1 title

Überschrift/Titel/Beschriftung der Daten

3.2 source_name

Bezeichnung der Datenurheber

3.3 source_url

  • URL unter der die Daten verfügbar sind
  • URL der Datenurheber
  • weitere relevante URLs

3.4 issued

Datum/Zeit der (Erst-)Publikation der Daten

3.5 modified

Datum/Zeit der letzten Aktualisierung der Daten

3.10 labels

Wie werden Merkmale und Werte einzelner Merkmale korrekt bezeichnet?

                       'group_2' = 'Geschlecht',
                       'n' = 'Anzahl Personen'),
     'group' = list('A' = '0 bis 5 Jahre',
                    'B' = '6 bis 10 Jahre'),
     'group_2' = list('Y' = 'weiblich',
                      'Z' = 'männlich'))

4 technische Repräsentation

Im Package biplaRmeta sind drei S6-Klassen definiert. Damit werden die Metainformationen abgebildet.

Zudem ist definiert, welches Feld die Informationen in welcher Form abspeichert:

field class depth length
title LanguageString 1 1
source_name LanguageString 1 Inf
source_url character 1 Inf
issued POSIXct 1 1
modified POSIXct 1 1
units LanguageString 2 1
aggregate character 2 1
dim_order character 1 Inf
hierarchy character Inf Inf
labels LanguageString 3 1
description LanguageString 3 Inf

4.1 MetaInfo

Diese Klasse repräsentiert die Metainformationen für eine Dateneinheit (z.B. eine Datentabelle).

    • $modified

4.3 LanguageString

Diese Klasse repräsentiert einen oder mehrere sprachliche Ausdrücke in mehreren Sprachen.

#> 2   en  English
#> 3   fr Français
#> 4   it Italiano

  • methods
